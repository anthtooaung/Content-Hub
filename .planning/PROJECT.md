# Content Hub

## What This Is

AI-powered social media content generator for the AI Solution Sprint Hackathon 2026. Users describe their business/campaign and get platform-specific social media posts, captions, hashtags, and CTAs generated by AI. Built for small-business owners and marketers who need content fast but don't have a content team.

## Core Value

Generate platform-specific social media content from a single campaign description — the user describes what they're promoting, picks their platforms, and gets ready-to-post content in seconds.

## Business Context

- **Customer**: Small-business owners, solo marketers, startup founders
- **Revenue model**: Hackathon demo — free to use
- **Success metric**: User can go from idea to ready-to-post content in under 2 minutes
- **Strategy notes**: Part of AI Solution Sprint Hackathon 2026

## Requirements

### Validated

- ✓ AI content generation (OpenAI primary, Gemini fallback) — existing
- ✓ Prisma + PostgreSQL database — existing
- ✓ NextAuth authentication — existing
- ✓ API routes for generate and history — existing

### Active

- [ ] Landing page with product-first hero, how-it-works, testimonials
- [ ] Auth flow (sign up, login, logout) with protected routes
- [ ] Generate form: wizard flow (form → loading → results) for TikTok, Instagram, Facebook
- [ ] Loading states: progress overlay with status messages and progress bar
- [ ] Result cards with platform badges, copy buttons, hashtags, CTAs
- [ ] Dashboard with timeline-based saved content history
- [ ] Empty state for dashboard when no content saved yet
- [ ] Save generated content to database
- [ ] Responsive design (desktop-first, mobile-friendly)

### Out of Scope

- OAuth login — email/password sufficient for hackathon
- Real-time collaboration — single-user tool
- Content scheduling/publishing — generation only
- Analytics/reporting — not in v1
- Custom branding/themes — default design system
- Multi-language support — English only

## Context

- **Existing codebase**: Next.js 14 App Router, TypeScript, Tailwind CSS, Prisma, NextAuth, OpenAI/Gemini
- **Current state**: Pages exist (landing, generate, dashboard) but use default Next.js template styling — needs full UI overhaul to match sketch findings
- **Design system**: From 4 sketch sessions — product-first hero, wizard generate flow, timeline dashboard, progress overlay loading states
- **Design tokens**: Blue primary (#2563EB), slate surfaces (#F8FAFC page, #FFFFFF cards), platform colors (TikTok #111827, Instagram #E1306C, Facebook #1877F2), Inter typeface, 8px spacing base
- **Platforms**: TikTok, Instagram, Facebook only for v1

## Constraints

- **Timeline**: Hackathon — ship fast, polish later
- **Stack**: Next.js 14, TypeScript, Tailwind CSS, Prisma, PostgreSQL — already chosen and in use
- **Auth**: Must require login — users need accounts to save content
- **LLM**: OpenAI primary, Gemini fallback — dual-model failover pattern already implemented

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Wizard flow for generation | Most natural for first-time users, one clear action at a time | ✓ Good |
| Timeline dashboard over grid/list | Groups content by time, natural for finding "that campaign from last week" | ✓ Good |
| Progress overlay over skeleton-only | Clearly communicates "something is happening" with status messages | ✓ Good |
| Product-first hero over generic SaaS | Shows actual product immediately — users understand in 3 seconds | ✓ Good |
| Auth required for hackathon | Users need to save content to history, builds user accounts | — Pending |
| 3 platforms only (TT/IG/FB) | Tight scope, matches sketches, hackathon timeline | ✓ Good |
