<task type="auto">
  <name>Task 2: Add prefers-reduced-motion support to globals.css</name>
  <files>app/globals.css</files>
  <action>
Add a prefers-reduced-motion media query to app/globals.css. Append the following block after the existing @layer utilities section:

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

This ensures users who have enabled reduced motion in their OS settings see no animations or transitions. The hover lift on dashboard cards (hover:-translate-y-1) will be suppressed because the transition duration is effectively zero. The spinner animation on the loading state will also be suppressed.

Do not modify any other part of globals.css. The existing CSS custom properties, Tailwind directives, and base styles remain unchanged.
