Implement Google and Facebook OAuth sign-in, account linking, and profile management page.

## Requirements

- Google OAuth sign-in via NextAuth GoogleProvider

- Profile page at `/profile` to view/manage linked accounts
- Account linking: connect multiple platforms to one account
- Account deletion with data cleanup
- Keep existing email/password login as fallback

## Technical Notes

### Database Schema

The `social_connections` table already exists in the schema:
```prisma
model social_connections {
  id                String              @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  user_id           String              @db.Uuid
  provider          oauth_provider_type
  provider_user_id  String?             @db.VarChar(255)
  provider_username String?             @db.VarChar(255)
  access_token      String?
  refresh_token     String?
  token_expires_at  DateTime?           @db.Timestamptz()
  platform_page_id  String?             @db.VarChar(255)
  used_for_login    Boolean             @default(false)
  used_for_posting  Boolean             @default(false)
  is_active         Boolean             @default(true)
  connected_at      DateTime            @default(now()) @db.Timestamptz()
  updated_at        DateTime            @default(now()) @updatedAt @db.Timestamptz()
  ...
}
```

This table is designed for storing OAuth tokens — no schema changes needed.

### OAuth Flow

1. User clicks "Sign in with Google"
2. NextAuth redirects to Google consent screen
3. Google redirects back with auth code
4. NextAuth exchanges code for tokens
5. `signIn` callback stores tokens in `social_connections`
6. User session created, redirected to `/generate`

### Account Linking

For linking additional accounts to an existing user:
1. User is already signed in
2. User clicks "Connect Facebook" on profile page
3. NextAuth handles OAuth flow
4. `linkAccount` callback creates new `social_connections` record
5. Profile page refreshes to show linked account
